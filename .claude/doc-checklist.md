# 项目文档清单

> 该清单记录 Quant 项目所有文档的位置与说明。
> **发版/文档更新前必须逐一核查，确保所有文档同步更新。**

---

## 一、核心文档（发版必查）

| 文件 | 说明 | 更新规则 |
|:---|---|:---|
| `CONSTITUTION.md` | 项目底层约束与规则 | 改规则/改风控参数/升版本号时更新 |
| `README.md` | 项目概述、版本号、快速开始、项目结构、路线图 | 每次发版必须更新版本号和路线图 |
| `CHANGELOG.md` | 版本变更日志 | 每次发版必须新增条目 |

---

## 二、docs/ 文档（涉及相关功能时更新）

| 文件 | 说明 | 更新条件 |
|:---|---|:---|
| `docs/使用指南.md` | 命令速查、参数说明、预期输出、FAQ | 新增/修改脚本参数、删除 Jupyter 引用、改配置项 |
| `docs/工程化改造.md` | 架构改造记录、版本规划表、落地优先级 | 新版本发版、架构变更、改造阶段完成 |
| `docs/交易频率影响分析.md` | Swing 定位论证 | 极少修改，策略定位变更时更新 |
| `docs/交易逻辑说明.md` | 因子分类、买卖点选择、风控说明 | 增删因子、改交易逻辑、改风控规则 |
| `docs/实验手册.md` | 实验记录、模型参数、训练输出解读 | 新增实验、改模型参数 |
| `docs/当前记录与更新计划.md` | 项目状态、实验结论、实盘就绪评估 | 新实验结论、阶段状态变更 |
| `docs/审查报告.md` | 代码审查详细结果 | 做完整代码审查时更新 |
| `docs/TO_DO_LIST.md` | 模拟盘 Sprint 计划与并行改造任务 | 完成任务/新增任务/改版本规划时更新 |

## 发版前检查清单

```markdown
- [ ] 版本号已更新？src/__init__.py + pyproject.toml
- [ ] CHANGELOG.md 已添加新版本条目？
- [ ] README.md 版本号 + 路线图已同步？
- [ ] CONSTITUTION.md 当前阶段已更新？
- [ ] 涉及 docs/ 下的哪些文件？逐一检查
- [ ] 新增/修改了脚本参数？→ 更新 docs/使用指南.md
- [ ] 新增/修改了配置项？→ 更新 .env.example + docs/使用指南.md
- [ ] 改动了架构？→ 更新 docs/工程化改造.md + README 项目结构
- [ ] 改动了因子/交易逻辑？→ 更新 docs/交易逻辑说明.md
- [ ] 改动了风控参数？→ 更新 CONSTITUTION.md + 交易逻辑说明.md
- [ ] 完成了任务/改动了版本规划？→ 更新 docs/TO_DO_LIST.md
- [ ] 所有 .md 文件无过时引用、无 typo
```

---

## 文档层级关系

```
CONSTITUTION.md              ← 最高优先级，不可变更规则
├── README.md                ← 项目概述
├── CHANGELOG.md             ← 版本变更
├── docs/使用指南.md          ← 操作说明
├── docs/工程化改造.md         ← 架构决策
├── docs/交易逻辑说明.md       ← 策略逻辑
├── docs/TO_DO_LIST.md       ← 实施计划
└── docs/审查报告.md          ← 审查记录
```

每次发版前，打开这个清单，从上到下逐项检查。
