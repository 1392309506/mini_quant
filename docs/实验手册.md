# 实验手册

> 面向个人研究者：如何设计、运行、解读量化实验。
> 本手册记录 Quant Trading System 的标准实验流程。

---

## 一、什么是实验

一个实验（experiment）是一次完整的 **训练 + 回测** 记录，包含：

```
实验 ID: 20260705_113346  （自动生成，格式 YYYYMMDD_HHMMSS）
├── config.json              ← 训练参数（特征、超参数、数据范围）
├── summary.json             ← 摘要指标（平均验证 RMSE）
├── windows_metrics.csv      ← 各 Walk-Forward 窗口的指标
├── feature_importance.csv   ← 特征重要性（按窗口）
├── oos_predictions.parquet  ← 各窗口的样本外预测
├── test_predictions.parquet ← 最终模型在测试集的预测
├── models/                  ← LightGBM 模型文件（W1.txt, W2.txt, ..., final.txt）
└── backtest/                ← 回测报告（自动生成，通过 run_backtest.py）
    ├── bt_config.json       ← 回测参数
    ├── bt_summary.json      ← 回测指标
    ├── equity_curve.png     ← 净值曲线
    ├── drawdown.png         ← 回撤曲线
    └── pred_vs_actual.png   ← 预测 vs 实际散点图
```

实验目录保存在 `experiments/<exp_id>/`，被 `.gitignore` 忽略，不入库。

## 二、命令速查

### 训练

| 命令 | 说明 |
|---|---|
| `python train_model.py` | 全量训练（28 个标的，全部窗口） |
| `python train_model.py --quick` | 快速训练（仅 5 个标的，用于调试） |
| `python train_model.py --target 21` | 预测 `forward_return_21`（默认是 10） |
| `python train_model.py --load <exp_id>` | 加载已有实验查看摘要 |

### 回测

| 命令 | 说明 |
|---|---|
| `python run_backtest.py <exp_id>` | 对实验运行回测 |
| `python run_backtest.py <exp_id> --top 10` | 每期最多持仓 10 只 |
| `python run_backtest.py <exp_id> --no-regime` | 关闭市场状态过滤器 |
| `python run_backtest.py` | 列出所有可用实验 |

### 数据与因子

| 命令 | 说明 |
|---|---|
| `python data_fetcher.py` | 增量更新数据 |
| `python factor_engine.py --cache` | 用缓存数据计算因子 |

## 三、标准实验流程

```
Step 1: 数据更新
  python data_fetcher.py --force
  → 更新 data/market_data.parquet

Step 2: 训练
  python train_model.py [--quick] [--target 21]
  → 生成 experiments/<新exp_id>/

Step 3: 查看训练结果
  python train_model.py --load <exp_id>
  → 打印 RMSE、特征重要性、OOS 预测统计

Step 4: 回测
  python run_backtest.py <exp_id>
  → 生成 experiments/<exp_id>/backtest/

Step 5: 分析
  查看 bt_summary.json（关键指标）
  查看 equity_curve.png（净值曲线）
  查看 feature_importance.csv（哪些因子有效）
```

## 四、实验参数速查

### 训练参数（`src/models/config.py`）

```python
# 特征列 — 增删因子只需改这里
FEATURE_COLS = ["MOMO_20", "MOMO_60", "MOM_RATIO", "RSI_14",
                "BB_POS", "VOL_MA_RATIO", "ATR_20_NORM", "VOLATILITY_20"]

# 预测目标 — 改变预测周期
DEFAULT_TARGET = "forward_return_10"   # 或 5 / 21

# LightGBM 核心参数
DEFAULT_LGBM_PARAMS = {
    "num_leaves": 31,          # 树的复杂度（越大越容易过拟合）
    "learning_rate": 0.05,     # 学习率（越小越稳定）
    "min_data_in_leaf": 50,    # 叶子节点最小样本数（越大越保守）
}
NUM_BOOST_ROUND = 1000         # 最大迭代次数
EARLY_STOPPING_ROUNDS = 50     # 早停轮数（验证集 N 轮不提升就停）

# Walk-Forward 窗口（年）
WF_INITIAL_TRAIN_YEARS = 6.0   # 初始训练集
WF_VAL_YEARS = 1.0             # 每个验证集
WF_STEP_YEARS = 1.0            # 滚动步长
WF_TEST_YEARS = 1.5            # 最终测试集
```

### 回测参数（`src/backtest/config.py`）

```python
TOP_K = 5                     # 每期最多持仓数
MIN_HOLD = 5                  # 最短持有天数
MAX_HOLD = 30                 # 最长持有天数（强制平仓）
ONE_WAY_FEE = 0.001           # 单边交易成本（含滑点）
INITIAL_CASH = 100_000        # 初始资金
USE_REGIME_FILTER = True      # 启用 SPY>MA200 过滤器
```

## 五、解读训练输出

### 5.1 窗口指标（`windows_metrics.csv`）

```
窗口     Train RMSE    Val RMSE    Best Iter
W1       0.0586        0.0591       3         ← 模型在学习！
W2       0.0583        0.0763       9         ← W2 (2022 加息年) 仍偏高
W3       0.0589        0.0537       53        ← 正常年份效果好
```

**怎么看：**
- **Best Iter > 1**：模型确实在迭代学习中（对比旧参数 best_iter=1）。值越高说明模型从训练中学到了更多。
- **W2 的 Val RMSE 偏高**（2022 年）：美联储加息导致市场波动加剧，因子的预测能力下降。这是正常的市场周期现象。
- **pred std 提升**：旧参数 pred std=0.0011，新参数+横截面特征后提升到 0.0022（2 倍），说明模型开始能区分股票间的差异。

### 5.2 特征重要性（`feature_importance.csv`）

```
feature        gain    split
ATR_20_NORM    2.60    124    ← 最重要的特征
MOM_RATIO      1.50    98
MOMO_60        1.42    85
VOLATILITY_20  1.16    72
RSI_14         0.23    12     ← 几乎没用
MOMO_20        0.14    8
BB_POS         0.00    0      ← 完全没用！
VOL_MA_RATIO   0.00    0
```

**怎么看：**
- **gain**：该特征对模型的总贡献。数值越大越重要。ATR_20_NORM 贡献了 2.60，说明波动率是最强的预测因子。
- **split**：该特征被用于分裂的次数。如果某个特征 gain 和 split 都是 0，说明它在预测 `forward_return_10` 时完全没用——可以考虑移除。
- **结论**：当前 8 个因子中，有效的是波动率（ATR_20_NORM, VOLATILITY_20）和动量（MOMO_60, MOM_RATIO）。均值回归类因子（RSI_14, BB_POS）和成交量因子（VOL_MA_RATIO）几乎没有预测能力。

### 5.3 OOS 预测统计

```
OOS 预测统计: 21168 行
  pred mean=0.0084        ← 平均预测 10 日后涨 0.84%
  pred std=0.0011         ← ⚠️ 预测标准差极小！
  actual mean=0.0052      ← 实际平均涨 0.52%
  actual std=0.0637       ← 实际标准差 6.37%
```

**怎么看：**
- **pred std << actual std**（差 60 倍）：这是**核心瓶颈**。模型对所有股票给出了几乎相同的预测值，无法区分强弱。说明模型只学到了"市场平均走势"，没学到"哪只股票比哪只涨得多"。
- **pred mean ≈ actual mean**：方向正确。模型知道 10 日后市场整体大概率上涨。
- **改进方向**：添加横截面特征（排名、Z-score）、调整模型参数让模型学得更细。

### 5.4 Rank IC

Rank IC 衡量模型对股票排序的准确性（-1 ~ 1）：

| Rank IC 均值 | 含义 |
|---|---|
| 0.05~0.10 | 弱但有效的排序能力（典型量化因子水平） |
| 0.10~0.20 | 中等强度的排序能力 |
| > 0.20 | 强排序能力 |
| ~ 0.00 | **完全随机** ← 当前状态 |

当前 Rank IC ≈ 0.005，基本等于随机猜测。

## 六、解读回测输出

### 6.1 关键指标（`bt_summary.json`）

```json
{
  "total_return_pct": 0.24,     // 3 年总收益 0.24%
  "annualized_return": 0.0002,  // 年化 0.02%
  "sharpe_ratio": 0.68,         // 夏普 < 1 说明收益风险比不理想
  "max_drawdown_pct": -0.07,    // 最大回撤 -0.07%
  "num_trades": 0,              // ⚠️ 交易次数为 0 说明信号生成有问题
  "win_rate": 0.0,
  "profit_factor": 0.0
}
```

**怎么看：**
- **total_return_pct < 5%**：策略基本没赚钱。原因是预测没有横截面区分度，导致每期选出的 top-k 股票和随机选差不多。
- **num_trades = 0**：要么是 `min_pred` 阈值太高（所有预测都低于阈值），要么是信号对齐出问题。检查 OOS 预测中 `pred > 0` 的比例。
- **max_drawdown 极小但 return 也极小**：策略几乎全程持有现金，不暴露风险也不赚钱。

### 6.2 图表解读

| 图表文件 | 怎么看 |
|---|---|
| `equity_curve.png` | 净值曲线应该稳定向上。如果平坦如直线 → 策略没交易。如果大起大落 → 风控不足。 |
| `drawdown.png` | 回撤应该在 20% 以内。如果持续创新低 → 策略失效。 |
| `pred_vs_actual.png` | 散点应该沿对角线分布。如果是一条水平线 → 预测没有区分度。如果散点图是圆形 → 相关性接近 0。 |

## 七、⚠️ 已知问题与排查

### 问题 1: Best Iteration = 1（第一轮就早停）

**原因**：学习率太高 + 特征太弱。模型在第一轮就达到了验证集上的最佳表现。

**解决（已实施）**：
1. 降低 `learning_rate`（0.05 → 0.01）
2. 增加 `early_stopping_rounds`（50 → 200）
3. 减少 `min_data_in_leaf`（50 → 10）让模型学得更细
4. 增加 `num_leaves`（31 → 63）提高模型复杂度

**效果**：best_iter 从 1 提升到 3/9/53（不同窗口不等），模型开始真正学习。

### 问题 2: 预测标准差太小（所有股票预测值接近）

**原因**：模型学会了"市场平均收益"，没学会"横截面差异"。

**解决（已实施）**：
1. ✅ 添加横截面特征（排名 + Z-score，共 16 个新特征）
2. ✅ 减小 `min_data_in_leaf`（50 → 10）
3. ✅ 增加 `num_leaves`（31 → 63）

**效果**：pred std 从 0.0011 → 0.0022（2 倍提升）。横截面特征（MOMO_60_zscore、ATR_20_NORM_zscore）现在占据特征重要性前 3 名。

**仍然不足**：pred std 0.0022 vs actual std 0.0637（差 29 倍）。需要更多横截面信息和更强的特征工程。

### 问题 3: 回测交易次数为 0

**原因**：信号生成逻辑与预测数据不匹配。

**排查步骤**：
1. 检查 `pred_oos` 中 `pred > 0` 的比例：`pred_oos[pred_oos['pred'] > 0].shape`
2. 检查调仓日历是否覆盖了预测日期范围
3. 检查 `min_pred` 是否设置过高

### 问题 4: 回测收益为正但跑输基准

**原因**：模型方向判断正确（知道市场涨）但选股能力不足。

**解决**：首要任务是提高横截面区分度，不是调整回测参数。

## 八、实验迭代模板

### 快速调试迭代

```bash
# 1. 调参后快速测试（5 只股票）
python train_model.py --quick

# 2. 看关键指标
python train_model.py --load <上一步的实验ID>

# 3. 如果指标有改善，跑全量
python train_model.py

# 4. 回测验证
python run_backtest.py <全量实验ID>
```

### 典型调参组合

| 场景 | `learning_rate` | `num_leaves` | `min_data_in_leaf` | `early_stopping_rounds` |
|---|---|---|---|---|
| 保守（当前） | 0.05 | 31 | 50 | 50 |
| 激进（推荐） | 0.01 | 63 | 10 | 200 |
| 中等 | 0.03 | 45 | 20 | 100 |

## 九、数据流简图

```
universe.txt
    ↓
data/market_data.parquet      ← 原始行情（OHLCV × 28 ticker）
    ↓
factor_panel                  ← 8 个因子（ticker × factor）
    ↓
训练数据 (Date,ticker)          ← 堆叠 + 远期收益标签 + dropna
    ↓
LightGBM Walk-Forward          ← 滚动训练 + 早停
    ↓
OOS 预测 → 回测 → 报告
```
