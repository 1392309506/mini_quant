# 实验手册

> 面向个人研究者：如何设计、运行、解读量化实验。
> 本手册记录 Quant Trading System 的标准实验流程。

---

## 一、什么是实验

一个实验（experiment）是一次完整的 **训练 + 回测** 记录，包含：

```
实验 ID: 20260705_113346  （自动生成，格式 YYYYMMDD_HHMMSS）
├── config.json              ← 训练参数（特征、超参数、数据范围）
├── summary.json             ← 摘要指标（平均验证 RMSE）
├── windows_metrics.csv      ← 各 Walk-Forward 窗口的指标
├── feature_importance.csv   ← 特征重要性（按窗口）
├── oos_predictions.parquet  ← 各窗口的样本外预测
├── test_predictions.parquet ← 最终模型在测试集的预测
├── models/                  ← LightGBM 模型文件（W1.txt, W2.txt, ..., final.txt）
└── backtest/                ← 回测报告（自动生成，通过 run_backtest.py）
    ├── bt_config.json       ← 回测参数
    ├── bt_summary.json      ← 回测指标
    ├── equity_curve.png     ← 净值曲线
    ├── drawdown.png         ← 回撤曲线
    └── pred_vs_actual.png   ← 预测 vs 实际散点图
```

实验目录保存在 `experiments/<exp_id>/`，被 `.gitignore` 忽略，不入库。

## 二、命令速查

### 训练

| 命令 | 说明 |
|---|---|
| `python train_model.py` | 全量训练（28 个标的，全部窗口） |
| `python train_model.py --quick` | 快速训练（仅 5 个标的，用于调试） |
| `python train_model.py --target 21` | 预测 `forward_return_21`（默认是 10） |
| `python train_model.py --load <exp_id>` | 加载已有实验查看摘要 |

### 回测

| 命令 | 说明 |
|---|---|
| `python run_backtest.py <exp_id>` | 对实验运行回测 |
| `python run_backtest.py <exp_id> --top 10` | 每期最多持仓 10 只 |
| `python run_backtest.py <exp_id> --no-regime` | 关闭市场状态过滤器 |
| `python run_backtest.py` | 列出所有可用实验 |

### 数据与因子

| 命令 | 说明 |
|---|---|
| `python data_fetcher.py` | 增量更新数据 |
| `python factor_engine.py --cache` | 用缓存数据计算因子 |

## 三、标准实验流程

```
Step 1: 数据更新
  python data_fetcher.py --force
  → 更新 data/market_data.parquet

Step 2: 训练
  python train_model.py [--quick] [--target 21]
  → 生成 experiments/<新exp_id>/

Step 3: 查看训练结果
  python train_model.py --load <exp_id>
  → 打印 RMSE、特征重要性、OOS 预测统计

Step 4: 回测
  python run_backtest.py <exp_id>
  → 生成 experiments/<exp_id>/backtest/

Step 5: 分析
  查看 bt_summary.json（关键指标）
  查看 equity_curve.png（净值曲线）
  查看 feature_importance.csv（哪些因子有效）
```

## 四、实验参数速查

### 训练参数（`src/models/config.py`）

```python
# 特征列 — 增删因子只需改这里（当前 13 个因子 → 39 个特征含横截面）
FEATURE_COLS = ["MOMO_20", "MOMO_60", "MOM_RATIO", "RSI_14",
                "BB_POS", "VOL_MA_RATIO", "ATR_20_NORM", "VOLATILITY_20",
                "BB_WIDTH", "HIGH_LOW_RATIO",
                "CHAIKIN_MF", "ULCER_INDEX", "MAX_DD_60",
                # + 每个因子的 _rank 和 _zscore 横截面版本]

# 预测目标 — 改变预测周期
DEFAULT_TARGET = "forward_return_10"   # 或 5 / 21

# LightGBM 核心参数（当前配置文件中的值）
DEFAULT_LGBM_PARAMS = {
    "num_leaves": 31,          # 树的复杂度（越大越容易过拟合）
    "learning_rate": 0.05,     # 学习率（越小越稳定）
    "min_data_in_leaf": 20,    # 叶子节点最小样本数
}
NUM_BOOST_ROUND = 3000         # 最大迭代次数
EARLY_STOPPING_ROUNDS = 500    # 早停轮数（验证集 N 轮不提升就停）

# Walk-Forward 窗口（年）
WF_INITIAL_TRAIN_YEARS = 6.0   # 初始训练集
WF_VAL_YEARS = 1.0             # 每个验证集
WF_STEP_YEARS = 1.0            # 滚动步长
WF_TEST_YEARS = 1.5            # 最终测试集
```

### 回测参数（`src/backtest/config.py`）

```python
TOP_K = 2                     # 每期最多持仓数
MIN_HOLD = 2                  # 最短持有天数（CONSTITUTION 建议 5，实验表明 2 可接受）
MAX_HOLD = 30                 # 最长持有天数（强制平仓）
MAX_REBALANCES_PER_WEEK = 5   # 每周调仓次数上限

POSITION_SCALE = 50.0         # 杠杆倍数（实验性参数，见风险警示）
USE_REGIME_FILTER = False     # SPY>MA200 过滤器（当前关闭）
```

## 五、解读训练输出

### 5.1 窗口指标（`windows_metrics.csv`）

```
窗口     Train RMSE    Val RMSE    Best Iter
W1       0.0588        0.0597       41        ← 模型稳定学习
W2       0.0606        0.0724       134       ← W2 (2022 加息年) Val 偏高，但模型仍学到更多
W3       0.0587        0.0585       68        ← 正常年份效果好
```

**怎么看：**
- **Best Iter > 1**：模型确实在迭代学习。值越高说明从训练中学到了更多（旧版本 best_iter=1 已修复）。
- **W2 的 Val RMSE 偏高**（2022 年）：美联储加息导致市场波动加剧，因子预测能力下降。这是正常的市场周期现象。
- **pred std 提升**：添加横截面特征后 pred std 从 0.0011 提升到 0.0022~0.0031，模型开始能区分股票间的差异。

### 5.2 特征重要性（`feature_importance.csv`）

```
feature               gain    split
ATR_20_NORM          12.25    215    ← 最重要的特征（波动率）
HIGH_LOW_RATIO       11.88    195    ← 新因子，排第 2
MOMO_60_zscore        7.22    142    ← 横截面排名版本更重要
MOMO_60               6.73    128
VOLATILITY_20         5.07    98
MOMO_20               4.61    85
```

**怎么看：**
- **gain**：该特征对模型的总贡献。数值越大越重要。
- **横截面特征（_zscore / _rank）**：比原始因子更重要，说明股票间的相对强弱比绝对值更关键。
- **波动率类因子（ATR_20_NORM, VOLATILITY_20）和动量类（MOMO_60）**是最强预测信号。
- **新因子 HIGH_LOW_RATIO** 直接排到第 2，说明振幅信息有独立贡献。
- 当前共 13 个基础因子（含 ULCER_INDEX, MAX_DD_60, CHAIKIN_MF）

### 5.3 OOS 预测统计

```
OOS 预测统计: 21168 行
  pred mean=0.0174        ← 平均预测 21 日后涨 1.74%
  pred std=0.0031         ← 预测标准差（有区分度，但仍有提升空间）
  actual mean=0.0089      ← 实际平均涨 0.89%
  actual std=0.0637       ← 实际标准差 6.37%
```

**怎么看：**
- **pred mean > 0**：方向正确，模型知道市场整体大概率上涨。
- **pred std 提升**：从最初的 0.0011 提升到 0.0031（3 倍），横截面特征确实帮助模型区分股票。
- **pred std < actual std**：仍有差距，模型对股票间差异的捕捉还不够强。
- **改进方向**：更多横截面特征、更多因子、更精细的超参数调优。

### 5.4 Rank IC

Rank IC 衡量模型对股票排序的准确性（-1 ~ 1）：

| Rank IC 均值 | 含义 |
|---|---|
| 0.05~0.10 | 弱但有效的排序能力（典型量化因子水平） |
| 0.10~0.20 | 中等强度的排序能力 |
| > 0.20 | 强排序能力 |
| ~ 0.00 | 完全随机 |

当前 Rank IC（旧模型 28只，fwd_21）≈ 0.03~0.05，处于"弱但有效"区间。叠加杠杆后回测夏普可达 5~9，说明即使是弱排序能力，通过杠杆放大也能产生可观收益。

## 六、解读回测输出

### 6.1 关键指标（`bt_summary.json`）

```json
{
  "total_return_pct": 0.24,     // 3 年总收益 0.24%
  "annualized_return": 0.0002,  // 年化 0.02%
  "sharpe_ratio": 0.68,         // 夏普 < 1 说明收益风险比不理想
  "max_drawdown_pct": -0.07,    // 最大回撤 -0.07%
  "num_trades": 0,              // ⚠️ 交易次数为 0 说明信号生成有问题
  "win_rate": 0.0,
  "profit_factor": 0.0
}
```

**怎么看：**
- **total_return_pct < 5%**：策略基本没赚钱。原因是预测没有横截面区分度，导致每期选出的 top-k 股票和随机选差不多。
- **num_trades = 0**：要么是 `min_pred` 阈值太高（所有预测都低于阈值），要么是信号对齐出问题。检查 OOS 预测中 `pred > 0` 的比例。
- **max_drawdown 极小但 return 也极小**：策略几乎全程持有现金，不暴露风险也不赚钱。

### 6.2 图表解读

| 图表文件 | 怎么看 |
|---|---|
| `equity_curve.png` | 净值曲线应该稳定向上。如果平坦如直线 → 策略没交易。如果大起大落 → 风控不足。 |
| `drawdown.png` | 回撤应该在 20% 以内。如果持续创新低 → 策略失效。 |
| `pred_vs_actual.png` | 散点应该沿对角线分布。如果是一条水平线 → 预测没有区分度。如果散点图是圆形 → 相关性接近 0。 |

## 七、⚠️ 已知问题与排查

### 问题 1: Best Iteration = 1

**状态：✅ 已修复**

旧参数（learning_rate=0.05, early_stopping=50, min_data_in_leaf=50）下，模型在第一轮就早停。调整后（lr=0.01, es=200, min_child=10）best_iter 提升到 3~53。当前配置（lr=0.05, min_data_in_leaf=20, es=500）更激进，best_iter 可达 41~134。

### 问题 2: 预测标准差太小

**状态：✅ 已改善，仍有提升空间**

添加横截面特征（排名+Z-score，共 16 个）后：
- pred std: 0.0011 → **0.0031**（3 倍提升）
- 横截面特征占据特征重要性前 3 名
- 回测从几乎不赚钱 → 年化 64%~81%

**仍然不足**：pred std 0.0031 vs actual std 0.0637（差 20 倍）。需要：
- 更多横截面因子
- 更多独立信号源
- 可能的模型架构改进

### 问题 3: 回测交易次数为 0

**状态：✅ 已修复**

此前信号生成逻辑与预测数据不匹配（min_pred 阈值导致所有预测被过滤）。当前 `MIN_PRED = -999.0`（不限制），交易正常。

### 问题 4: 回测收益为正但跑输基准

**状态：✅ 不再是问题**

当前回测（p=50, 28 只）年化 64.75%，显著跑赢 SPY 同期（~12%）。夏普 5.80，说明收益风险比已远超简单持有。

### 风险警示

**高杠杆下的爆仓风险**：当前回测使用 position_scale=50 意味着每个仓位 = 账户的 25~50 倍。详见 [docs/当前记录与更新计划.md](当前记录与更新计划.md) 风险警示章节。

## 八、实验迭代模板

### 快速调试迭代

```bash
# 1. 调参后快速测试（5 只股票）
python train_model.py --quick

# 2. 看关键指标
python train_model.py --load <上一步的实验ID>

# 3. 如果指标有改善，跑全量
python train_model.py

# 4. 回测验证
python run_backtest.py <全量实验ID>
```

### 典型调参组合

| 场景 | `learning_rate` | `num_leaves` | `min_data_in_leaf` | `early_stopping_rounds` |
|---|---|---|---|---|
| 当前配置 | 0.05 | 31 | 20 | 500 |
| 激进（推荐） | 0.01 | 63 | 10 | 200 |
| 中等 | 0.03 | 45 | 20 | 100 |

回测收益: xx%
结论: ...
