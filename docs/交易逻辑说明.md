# 交易逻辑说明

> 本文档详细说明当前量化交易系统的完整交易逻辑。
> 最后更新: 2026-07-07 | 对应版本: v0.5.0

---

## 一、策略定位

### 1.1 策略类型

- **中低频 Swing 交易**：持仓数天到数周（`min_hold=2`, `max_hold=30`）
- **横截面多因子模型**：LightGBM 统一学习多只股票间的因子差异
- **做多策略**：仅持有多头仓位，不持有空头

### 1.2 数据基础

| 项目 | 说明 |
|------|------|
| 数据源 | yfinance（默认），Alpha Vantage（兜底） |
| 频率 | 日线（OHLCV） |
| 复权 | `Adj Close` 作为收盘价 |
| 标的池 | 固定列表（`universe.txt`），当前 V1=28 只 |
| 历史长度 | 约 10 年（2015-03 ~ 至今） |

## 二、因子系统

### 2.1 因子列表

当前 13 个因子，分为三大类：

**动量类（3 个）**

| 因子 | 计算公式 | 含义 |
|------|---------|------|
| `MOMO_20` | `close / close.shift(20) - 1` | 20 日价格动量 |
| `MOMO_60` | `close / close.shift(60) - 1` | 60 日价格动量 |
| `MOM_RATIO` | `MOMO_20 / (MOMO_60 + ε)` | 短期 vs 长期动量比 |

**均值回归类（3 个）**

| 因子 | 计算公式 | 含义 |
|------|---------|------|
| `RSI_14` | 14 日 RSI | 超买/超卖信号 |
| `BB_POS` | `(close - BB_lower) / (BB_upper - BB_lower)` | 布林带位置 |
| `VOL_MA_RATIO` | `volume / volume.rolling(20).mean()` | 成交量相对强弱 |

**波动率/风险类（6 个）**

| 因子 | 计算公式 | 含义 |
|------|---------|------|
| `ATR_20_NORM` | `ATR(20) / close` | 归一化平均真实波幅 |
| `VOLATILITY_20` | `close.pct_change().rolling(20).std()` | 20 日波动率 |
| `BB_WIDTH` | `(BB_upper - BB_lower) / close` | 布林带宽度（归一化） |
| `HIGH_LOW_RATIO` | `(high - low).rolling(20).mean() / close` | 振幅比 |
| `ULCER_INDEX` | 回撤深度 × 持续时间平方根 | 下行风险度量 |
| `MAX_DD_60` | 60 日最大回撤（负值） | 刚大跌过的股票 |

**资金流类（1 个）**

| 因子 | 计算公式 | 含义 |
|------|---------|------|
| `CHAIKIN_MF` | 佳庆资金流指标 | 资金流入/流出压力 |

### 2.2 因子组合方式

**不是手动加权，而是由 LightGBM 自动学习。**

每个因子先计算原始值，然后对每个日期做横截面变换：

```
原始因子(13 个)  →  rank(13 个)  →  zscore(13 个)
                     ↑ 截面排名归一化  ↑ 截面 Z-score 标准化
```

最终模型输入 = **39 个特征**（13 原始 + 13 rank + 13 zscore）。

LightGBM 自动学习每个特征的权重、非线性关系、交互作用。不需要手动调因子权重。

### 2.3 因子验证

每次 `daily_inference.py` 运行时自动执行因子验证：

- 检查每个因子的值是否在预设合理范围内
- 检查缺失值比例是否超过 30%
- 通过率 < 80% 时发出警告但不阻断运行
- 阈值见 `src/factors/validation.py`

## 三、模型

### 3.1 模型架构

| 项目 | 说明 |
|------|------|
| 框架 | LightGBM（GBDT） |
| 训练方式 | 横截面统一训练（pool 所有标的） |
| 验证方式 | Walk-Forward Validation |
| 目标变量 | `forward_return_{10,21}`（10 或 21 个交易日后的收益） |
| 超参数 | `num_leaves=31`, `learning_rate=0.05`, `feature_fraction=0.8` |

### 3.2 可用模型

| 版本 | 标的数 | 预测目标 | 特征数 | Val RMSE | 用途 |
|:---:|:---:|:---:|:---:|:---:|:---|
| V1 | 28 只 | fwd_21（21 天） | 30 | 0.0934 | 当前主力，数据稳定 |
| V2 | 118 只 | fwd_10（10 天） | 30 | 0.0622 | 需完整 118 只数据池 |

> **注意**：V2 模型训练时横截面 rank/zscore 基于 118 只计算，如果当前数据不足 118 只，预测值会偏移。当前统一使用 `--model V1`。

### 3.3 模型存储

```
models/
  V1_28stock_fwd21/          ← 版本目录
    manifest.json             ← 元信息（训练日期、特征列表、指标）
    models/
      final.txt               ← LightGBM 模型文件
      W1.txt, W2.txt, ...     ← Walk-Forward 各窗口模型
    feature_importance.csv
```

## 四、股票选择逻辑

### 4.1 选择流程

```
1. 计算所有 13 个因子 → 39 个特征
2. LightGBM 推理 → 每只股票一个预测分（pred）
3. 在每个调仓日，选出 pred 最高的 top_k 只股票
4. 过滤条件：pred > 0（正预测）
5. 可选：SPY 200 日均线过滤器（市场状态）
```

### 4.2 调仓日历

| 频率 | 说明 |
|:---:|:---|
| 每周 5 次 | 默认每个交易日都评估（max_per_week=5） |
| 每周 2 次 | 保守模式，周一和周四 |
| 每周 3 次 | 中庸模式，周一、三、五 |

当前默认：**每周 5 次**（每个交易日）。

### 4.3 具体示例

```
2026-07-07（收盘后计算）
  → 预测 28 只股票的未来收益排名
  → top 5 = MU, TSLA, UNH, WDC, WMT
  → 2026-07-08（开盘）执行买入
```

## 五、买卖点选择

### 5.1 入场

| 规则 | 值 |
|------|:---:|
| 信号发出 | 每个交易日收盘后（daily_inference.py） |
| 实际入场 | **次日开盘**（`open[t+1]`） |
| 禁止 | 用 `close[t]` 同时作信号和入场（前视偏差） |
| 仓位 | 等权分配：`size = 1 / top_k × position_scale` |

### 5.2 出场

| 规则 | 值 |
|------|:---:|
| 最小持有 | `min_hold=2` 天（*由执行层约束，不生成自动出场信号*） |
| 强制出场 | `max_hold=30` 个交易日后自动出场 |
| 出场价 | 强制出场日的开盘价 |

> **修复说明 v0.5.0**：旧逻辑在 min_hold 天自动生成出场信号，导致 max_hold 强制出场从未生效。现已修正：只有 max_hold 生成出场信号，min_hold 由执行层保证不提前平仓。

### 5.3 成本模型

| 费用项 | 费率 |
|:------|:---:|
| 佣金 | 0.1%（单边） |
| 滑点 | 0.05% |
| 点差 | 0.02% |
| **总单边成本** | **0.17%** |

## 六、杠杆选择

### 6.1 安全杠杆上限

基于 v0.5.0 验证结论：

| 方法 | 安全上限 |
|:---|:---:|
| 基于最差单日（-0.29% @ scale=1） | ≤ **7.0x** |
| 基于 3σ 日波动（0.03%/日） | ≤ **20.3x** |
| **推荐保守值** | **≤ 7.0x** |

### 6.2 杠杆计算方式

```
position_scale = 7（7x 杠杆）
top_k = 5
每只股票仓位 = 1/5 × 7 = 1.4x（账户的 1.4 倍）
最大总敞口 = 5 × 1.4 = 7.0x
```

### 6.3 为什么不是更高杠杆

高杠杆（如 50x）在回测中表现好，但实盘中：
- 1 个仓位反向 2% = 亏损 20%
- Exness CFD 保证金要求 1%~2%
- 亏损达保证金 50%~80% 即强制平仓
- **实际可承受亏损可能只有账户的 1%~2%**

## 七、风险管理

### 7.1 风控规则（代码自动执行）

| 规则 | 值 | 说明 |
|:---|:---:|:---|
| 组合杠杆上限 | **7.0x** | 总敞口（已有 + 新单）÷ 权益 |
| 单标的最大暴露 | **30%** | 单只股票占权益比 |
| 单笔止损 | **8%** | 浮动亏损达 8% 自动平仓 |
| 单日最大亏损 | **5%** | 当日亏损达 5% 停止交易 |
| 每日最大交易次数 | **4 次** | Swing 频率限制 |

### 7.2 风控执行流程

```
下单前检查（RiskManager.check_order）：
  1. 计算全组合杠杆（已有持仓 + 新订单）≤ 7x
  2. 单标的总暴露 ≤ 30%
  3. 当日交易次数 ≤ 4
  4. 当日亏损 ≤ 5%
  5. 单笔潜在亏损 ≤ 2%

持仓中检查（RiskManager.enforce_stop_loss）：
  1. 遍历所有持仓
  2. 计算浮动盈亏 %
  3. 亏损 ≥ 8% → 标记强制平仓

每日上限检查（RiskManager.check_daily_limits）：
  1. 交易次数达上限 → 停止新开仓
  2. 当日亏损达上限 → 停止所有交易
```

### 7.3 模拟/实盘模式

| 模式 | 默认 | 说明 |
|:---:|:---:|:---|
| 模拟 | ✅ 是 | `Broker(simulate=True)`，无需 MT5，本地可用 |
| 实盘 | ❌ 否 | 需 `.env` 配置 MT5 账号，显式切 `simulate=False` |

## 八、每日信号生成 Pipeline

```
daily_inference.py 流程：
  1. 加载模型（从 models/ registry）
  2. 拉取最新行情（缓存 ≤7 天则跳过）
  3. 检查标的池数量（防止 V2 模型在 28 只数据上运行）
  4. 计算 13 个因子
  5. 添加横截面特征（rank + zscore）
  6. LightGBM 推理 → 预测分
  7. 生成调仓日历（未来 90 天）
  8. 生成入场信号（top_k 选股）
  9. 生成出场信号（max_hold 强制出场）
  10. 保存信号到 data/signals/

输出文件：
  data/signals/
    {date}_entries.parquet    ← 入场信号（bool 矩阵）
    {date}_exits.parquet     ← 出场信号（bool 矩阵）
    latest_entries.parquet   ← 最新入场信号（覆盖）
    latest_exits.parquet     ← 最新出场信号（覆盖）
    latest_predictions.parquet ← 预测值
    latest_summary.json      ← 信号摘要
```
