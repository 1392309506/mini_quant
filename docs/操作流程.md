## 第二步：配置 .env

打开项目根目录的 `.env` 文件，按你的网络环境填写：

```ini
# 数据后端：yfinance（推荐，数据全）或 alpha_vantage（兜底，仅100天）
DATA_BACKEND=yfinance

# 代理配置（关键！yfinance 在中国直连会被限流）
# 填你 v2rayN/Clash 的 HTTP 代理端口，常见值：
#   v2rayN 默认:   http://127.0.0.1:10808
#   Clash 默认:    http://127.0.0.1:7890
HTTP_PROXY=http://127.0.0.1:10808
HTTPS_PROXY=http://127.0.0.1:10808

# Alpha Vantage Key（兜底用，免费注册: https://www.alphavantage.co/support/#api-key）
ALPHA_VANTAGE_KEY=你的KEY
```

**如何确认代理端口：** 打开你的代理软件（v2rayN/Clash），看「HTTP 代理」监听端口。确认代理软件**正在运行**，且开启「系统代理」或全局模式。

## 第五步：理解输出

### 数据完整性报告

```
检查: OK=22(85%), 需关注=4/26
  [V] 数据偏少: 420天 (<200，无法算 MA200)
  [BRK-B] 缺失值: 45个(2.3%)
```

- **数据偏少**：该股票上市较晚，不足 200 天。可从 `TRADE_UNIVERSE` 移除，或不依赖它的 MA200 因子。
- **异常波动 >±20%**：可能是财报跳空或拆股。少量可接受，大量需检查数据。

### 因子验证

```
[AAPL] MOMO_20: 范围异常 [-5.23, 0.45] (应在 [-0.5, 0.5])
```

通常源于：财报日跳空、拆股未还原、数据源错误。在 Jupyter 里定位具体日期排查。

## 常见问题排查

### Q1: yfinance 报 "Too Many Requests / Rate limited"

**原因：** 代理未开启，或代理端口填错。
**解决：**
1. 确认 v2rayN/Clash 正在运行
2. 在 `.env` 中确认 `HTTP_PROXY` 端口与代理软件一致
3. 测试代理是否通：
   ```bash
   curl -x http://127.0.0.1:10808 https://www.google.com -I
   ```

### Q2: 不想用代理，能用 AV 吗？

能，但**免费层 AV 只返回最近 100 天数据**，不足以计算 200 日均线因子。仅作兜底：

```ini
# .env
DATA_BACKEND=alpha_vantage
```

### Q3: "ModuleNotFoundError: No module named 'quant'"

从项目根目录运行脚本，Python 才能找到 `quant/` 包：
```bash
cd d:/Code_store/Quant
python data_fetcher.py
```

### Q4: 因子全是 NaN

某股票数据不足 60 天会被跳过。检查 `data/factor_validation.csv`，从 `TRADE_UNIVERSE`（在 `quant/fetcher.py`）移除问题标的。

## 高级用法（Jupyter Notebook）

```python
from quant.fetcher import fetch_all_data, extract_close_matrix, extract_volume_matrix
from quant.factor import build_factor_panel, rsi_14

df = fetch_all_data()
closes = extract_close_matrix(df)
volumes = extract_volume_matrix(df)

factors = build_factor_panel(closes, volumes)

# 查看 AAPL 最新 RSI
aapl_rsi = factors["AAPL"]["RSI_14"].dropna()
print(f"AAPL RSI: 最新={aapl_rsi.iloc[-1]:.1f}")

# 单因子分析
rsi = rsi_14(closes["AAPL"])
oversold = rsi[rsi < 30]   # 超卖信号
print(oversold.tail(5))
```
