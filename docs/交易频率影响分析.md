# 交易频率影响分析

> 交易频率像一把尺子，贯穿量化系统的每层决策。
> 本文以本项目的定位——**中低频 Swing（数天到数周）**——为主线，分析频率对系统设计的影响。

---

## 一、频率谱系总览

| 维度 | 高频（秒~分钟） | 中频（小时~天） | **中低频 Swing** | 低频（月~季） |
|---|---|---|---|---|
| 交易成本占比 | ⛔ 极重要——成本是最大敌人 | 重要 | **适度关注** | 不重要 |
| 数据需求 | Tick / 1min | 5min / 日线 | **日线就够了** | 日线 / 周线 |
| 信号来源 | 微观结构 / 订单流 | 日内动量/反转 | **技术因子 + 横截面 + 基本面** | 纯基本面 |
| 基础设施 | 托管服务器 / FPGA / 低延迟 | 可靠 VPS | **普通电脑 + 脚本** | 电子表格都行 |
| 过拟合风险 | 极高（成千上万次交易） | 高 | **中等（数百~数千笔）** | 低 |
| 资金容量 | 低 | 中等 | **大** | 极大 |
| 时间投入 | 全天监控 | 每天复盘 | **每周看几次** | 每月 |

**核心规律**：频率每下降一个数量级，系统的成本/数据/基础设施需求就简化一层，
但信号稀缺性和耐心要求反而更高。

---

## 二、对个人操作的影响

### ✅ 有利的

- **不需要盯盘**——下单后关掉都行，适合有主业的投资者
- **现有因子已经够用**——MOMO_20/60、RSI_14、BB_POS 都是 Swing 级别设计（20 天窗口不是巧合），不需要重做
- **基础设施零成本**——yfinance 日线 + parquet 缓存 + 笔记本跑全部够用
- **数据源简单**——日线 yfinance 免费就能拉，不需要购买高频数据

### ⚠️ 需要调适的

- **信号密度低**——一个月可能就 2~4 次交易，需要习惯长时间等待信号
- **回测周期要长**——Swing 需要 3~5 年+ 数据才有统计意义（高频一天就有 100 笔）
- **滑点心理影响**——一笔亏损可能就是几个月的收益，心理承受力要求更高

### 🚫 直接不需要做的

- 微结构因子（买卖不平衡、VWAP 偏离等）
- 低延迟执行（独享 VPS、API 优化到微秒）
- 日内数据清洗（时区对齐、聚合间隙）

## 三、对量化项目的设计影响

### 3.1 数据层

| 当前做法 | 对 Swing 合适吗？ |
|---|---|
| 日线 yfinance | ✅ **非常合适**——Swing 不需要更细的数据 |
| 7 天缓存 | ✅ 日线每天更新一次就够 |
| 118 个美股标的 | ✅ Swing 建议 20~50 个标的，118 只覆盖更广 |
| yfinance 数据已覆盖 5 年+ | ✅ Swing 至少需要 3 年+ 回测数据 |

**结论：数据层不需要改动。**

### 3.2 因子层

| 因子 | 对 Swing 的评价 |
|---|---|
| MOMO_20 / MOMO_60（20/60 天动量） | ✅ **核心因子**，Swing 趋势追踪主力 |
| RSI_14（14 天超买超卖） | ✅ 经典 Swing 信号 |
| BB_POS（20 天布林带位置） | ✅ 均值回归 Swing 入场 |
| ATR_20_NORM（20 天波动率） | ✅ 仓位管理用 |
| VOLATILITY_20（20 天收益率波动） | ✅ 同上 |
| MOM_RATIO（动量加速比） | ✅ Swing 趋势确认 |
| VOL_MA_RATIO（成交量/均量比） | ✅ 放量突破确认 |

**结论：因子层非常适合 Swing，不需要换方向。** 后续如果拓展，应优先加：

- **横截面因子**（相对强弱、板块排名）——Swing 更适合横截面选股而非时序择时
- **基本面过滤**（盈利增长、估值分位）——数周持仓足够让基本面信息体现

### 3.3 模型层（已实现）

Swing 对 ML 训练的影响：

| 维度 | Swing 要求 | 当前做法 |
|---|---|---|
| 预测周期 | 未来 5~20 个交易日的方向 | 标签用 `forward_return_{5,10,21}`，当前默认 fwd_10  ✅ |
| 样本量 | 5 年 × 118 标的 ≈ 15,000 条 | **横截面统一训练**（pool 所有标的）✅ |
| 评估周期 | 滚动 12 个月 | Walk-Forward 窗口，6y 训练 + 1y 验证 ✅ |

### 3.4 回测层

Swing 回测比高频好做，但有常见陷阱：

| 正确做法 | 错误做法 |
|---|---|
| 用 `open[t+1]` 作为入场价（次日开盘） | 用 `close[t]` 同时作信号和入场（未来函数） |
| 每笔手续费固定 + 滑点按 ATR 百分比估算 | 忽略滑点（Swing 滑点占比虽小但不能忽略） |
| 信号只允许每周 1~2 次调仓 | 信号每天变化就每天调仓（过度交易） |

**Swing 回测规则推荐：**
- 信号发出 → **次日开盘**执行
- 持仓最短 2 天，最长 30 天
- 交易成本：单边 0.17%（含佣金 0.1% + 滑点 0.05% + 点差 0.02%）
- 信号频率：每周最多 5 次（已从 2 次放宽）
- 样本起点：至少 3 年日线

### 3.5 执行层

回到 Exness 选型——Swing 频率下：

- **Raw Spread 和 Zero 实际差别不大**——一个月几笔交易，佣金差不了几块钱
- 更关键的问题是：**Exness 支持的 CFD 品种和 yfinance 的美股标的能否对齐？** Swing 最怕"回测跑 A、实盘做 B"

---

## 四、一句话总结

> Swing（数天~数周）意味着：**日线数据 + 技术因子 + 横截面模型 + 简单回测就够用。**
> 当前项目的基础层已天然适合 Swing，不需要为了"听起来高级"去加高频模块。
> 专注把 **因子 → 模型 → 回测** 这个链路做扎实，才是正确的路。
