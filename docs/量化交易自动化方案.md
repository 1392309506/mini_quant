## 1. 先讲三句实话

这不是泼冷水，是让你少走弯路。在你 AI 领域的经验里你会知道：**很多论文的结果在换数据集后无法复现**。量化交易也是一样——别人的策略你跑可能亏钱，跑赢大盘的概率远低于你的直觉。

### 实话一：1,000 美元的局限

- 买不了太多股票。按美股 $50-$200/股计，1 笔买入可能就用掉你 5%-20% 的资金。
- **用 CFD 的微型手（0.01 手）解决**。Exness / IC Markets 上，0.01 手 NAS100 ≈ $1.5-2/点，你可用 $50 控制一笔纳指交易。美股 CFD 也支持 0.01 手，相当于买几美元的面值。
- 佣金是固定成本。CFD 交易的点差（spread）是主要成本，0.01 手 NAS100 点差约 $0.5-1.5/次，比 ETF 佣金低得多。
- 结论：**1k 最适合做学习验证和跑通自动化流水线**。能盈利当然好，但首要目标是学会这套东西怎么运转。

### 实话二：量化的核心不是代码，是数学

AI 模型的代码任何人都会写，但**赚不赚钱取决于因子（features）的好坏和风控**。你 AI 领域的特征工程、防过拟合、交叉验证、out-of-sample 测试，这些经验在量化里完全适用，甚至更关键。

### 实话三：自动化会放大错误

手动交易下错单亏 100 块——心疼但可以停。
自动化下错单：**一个 bug 可以在 30 分钟内亏掉全部本金**。
所以下面的自动化方案里，**前 2 个月只做模拟盘回测 + Paper Trading（纸交易，模拟资金实时跑）**。不要怀疑这条规则——所有专业量化团队都有类似的 "kill switch" 文化。

## 2. 困难一：人不在中国，怎么买美股/黄金/纳指？

### ⚠️ 先说监管背景

你提到的老虎和富途被禁，是事实。2025 年 5-6 月，中国证监会再次明确了未经批准的跨境证券交易违规。棕榈资本、富途、老虎等均被点名。

那是不是就不能做了？不一定。**关键在于区分"证券"和"衍生品"——**

- **券商（IBKR / 老虎 / 富途）**：直接买卖美股证券，受中国证监会跨境监管关注
- **离岸 CFD 券商**：交易的是差价合约（Contract for Difference），属于衍生品交易，不在证监会直接管辖范围内，灰色但广泛存在

你做过 FTMO，上面用的就是 CFD 模式（外汇、指数、黄金都是 CFD）。所以这条路对你来说很熟悉。

### 2.1 币安为什么也不适合你

币安是加密货币交易所——它交易的是 BTC/ETH 等，不交易美股。它有 "Stock Tokens" 但已于 2022 年停止。所以币安的路对你也不通。

### 2.3 自动化怎么连：Python → MT5 → CFD 券商

```python
# 这是你将来连接券商并下单的方式
import MetaTrader5 as mt5

# 初始化 MT5 连接
mt5.initialize()

# 登录你的交易账户（Exness 开户后会给账号和密码）
account = 12345678

mt5.login(account, password, server)

# 获取账户信息
info = mt5.account_info()
print(f"余额: ${info.balance}")

# 下单一手 NAS100（纳指 CFD）
symbol = "NAS100"
request = {
    "action": mt5.TRADE_ACTION_DEAL,
    "symbol": symbol,
    "volume": 0.01,        # 0.01 手 = 迷你合约
    "type": mt5.ORDER_TYPE_BUY,
    "price": mt5.symbol_info_tick(symbol).ask,
    "sl": 19500.0,         # 止损
    "tp": 21000.0,         # 止盈
}
result = mt5.order_send(request)
print(f"订单结果: {result.comment}")
```

### 2.6 重要提醒：CFD 的风险

CFD 有杠杆，这是双刃剑：

```
你的 1,000 美元本金
        ↓
用 1:100 杠杆，你可以控制 100,000 美元的名义价值
        ↓
市场跌 1% → 你亏 1,000 美元 → 爆仓
        ↓
所以：小仓位（0.01 手）+ 止损（SL）是生存关键
```

你没有杠杆过量的风险，因为你做 swing 交易（不频繁交易），而且用 0.01 迷你手。但务必记得：**CFD 不是 ETF，它没有资产归属权，只有价格差额**。你的券商如果出问题（比如破产），你手里的 CFD 合约可能一文不值。所以选规模大、监管合规的券商（Exness 受 FCA / CySEC 监管）很重要。

### 3.4 因子（Features）设计

因子的设计直接决定策略是否赚钱。以下是适合小资金 swing 交易的因子：

#### A. 动量因子（买入趋势延续的股票）

| 因子名 | 计算方式 | 直觉 |
|--------|---------|------|
| MOMO_20 | 最近 20 个交易日的累计收益率 | 短期趋势 |
| MOMO_60 | 最近 60 个交易日的累计收益率 | 中期趋势 |
| MOM_RATIO | 短/长期动量比，`MOMO_20 / MOMO_60` | 趋势加速/减速 |

#### B. 均值回归因子（买入超卖的股票）

| 因子名 | 计算方式 | 直觉 |
|--------|---------|------|
| RSI_14 | 14 天相对强弱指标 | < 30 超卖，> 70 超买 |
| BB_POS | 布林带位置 `(close - BB_low) / (BB_high - BB_low)` | 接近 0 超卖，接近 1 超买 |
| VOL_MA_RATIO | 当前成交量 / 20日均量 | 异常放量可能是转折信号 |

#### C. 波动率因子

| 因子名 | 计算方式 | 直觉 |
|--------|---------|------|
| ATR_20 | 20 天平均真实波幅 | 衡量波动，分配仓位时参考 |
| VOLATILITY_20 | 20 天收益率标准差 | 同上，备选 |

#### D. 宏观环境过滤器（Market Regime Filter）

这是**最重要的风控机制**——不要在熊市里频繁做多：

```python
# 市场状态过滤器
spy = yf.download("SPY", start="2020-01-01", end="2025-12-31")
spy_ma200 = spy["Close"].rolling(200).mean()

def market_filter(date):
    """SPY 在 200 日均线上方才允许做多"""
    today_close = spy.loc[date, "Close"]
    today_ma200 = spy_ma200.loc[date]
    return today_close > today_ma200  # True = 允许做多
```

#### E. 宏观辅助因子（来源于 FRED）

| 因子 | 用途 |
|------|------|
| 10年期国债收益率 | 影响估值风格（成长 vs 价值） |
| VIX | 恐惧指数，高 VIX 时减少仓位 |
| 美元指数 DXY | 影响黄金 GLD 的价格 |

### 3.5 信号融合：用 ML 把因子组合在一起

你的 AI 背景在这步最值钱。方法很简单：

**回归角度**：
```
X = 所有因子值（每只股票每个时间步一行）
y = 未来 5 个交易日的收益率（Forward 5-day Return）
模型：LightGBM Regressor
预测得分 = 模型输出（越大越看好）
```

**分类角度**（新手更稳定）：
```
X = 所有因子值
y = 未来 5 日收益率 > 2%（二分类：1=涨超 2%，0=没到）
模型：LightGBM Classifier
预测得分 = 概率 p(y=1)
```

**关键的反过拟合措施（一定不能省！）：**

1. **时序切分，不是随机切分**：
   ```python
   # ❌ 错误：随机打乱
   train, test = train_test_split(data, shuffle=True)
   
   # ✅ 正确：按时间顺序切分
   train = data.loc[:"2023-12-31"]
   valid = data.loc["2024-01-01":"2024-06-30"]
   test  = data.loc["2024-07-01":]
   ```

2. **Walk-Forward 交叉验证**：
   ```python
   # 每 6 个月重新训练一次，用过去 2 年数据训练
   # 窗口滚动：2022-01~2023-12 训练 → 2024-01~2024-06 预测
   #          2022-07~2024-06 训练 → 2024-07~2024-12 预测
   ```

3. **特征重要性监控**：
   每次重新训练后记录 top-10 因子。如果某次训练因子重要性发生剧烈变化，**停止交易、检查原因**。

4. **过拟合检测**：
   ```python
   # 训练集和测试集的夏普比率差距超过 0.5 → 过拟合超标
   sharpe_train = train_returns.mean() / train_returns.std() * sqrt(252)
   sharpe_test  = test_returns.mean() / test_returns.std() * sqrt(252)
   if abs(sharpe_train - sharpe_test) > 0.5:
       print("⚠️ 过拟合超标，需要简化模型或增加数据")
   ```

**风险控制的几条铁律（违反任何一条就自动停止）：**

1. **单只仓位 ≤ 总资金的 20%**（用 1,000 元就是 ≤200 元）
2. **总敞口（所有持仓之和）≤ 总资金的 95%**（至少留 5% 现金）
3. **硬止损 -8%**：任何单只亏损超过 8%，无条件卖出
4. **最大回撤保护 -15%**：账户总净值从最高点回撤超过 15%，全部平仓、停止交易 1 周
5. **熊市不做多**：SPY 跌破 200 日均线时，清仓所有多头（可以保留 QQQ 和 GLD 用其他规则判断）

2. 市场过滤检查：
   if SPY 在 200 均线下方 and 持有的是纯多头：
      全部清仓，本周不交易

   scores = model.predict_proba(features_today)

   current_cash = broker.get_cash()

### 4.3 MT5 → CFD 券商自动化接入指南

你的策略计算出信号后，需要连接到券商执行下单。通过 `MetaTrader5` Python 库完成：

```python
import MetaTrader5 as mt5
import pandas as pd

# 1. 初始化 MT5
if not mt5.initialize():
    print("MT5 初始化失败")
    quit()

# 2. 登录交易账户
account = 12345678          # 你的 Exness/IC Markets 账号
password = "your_password"  # 交易密码（不是投资密码）
server = "Exness-MT5Trial"  # 服务器名（开户后提供）
mt5.login(account, password, server)

# 3. 获取账户信息
info = mt5.account_info()
print(f"余额: ${info.balance}, 净值: ${info.equity}")

# 4. 获取实时行情
tick = mt5.symbol_info_tick("NAS100")
print(f"NAS100 买价: {tick.bid}, 卖价: {tick.ask}")

# 5. 下单（以做多 NAS100 为例）
symbol = "NAS100"
point = mt5.symbol_info(symbol).point
request = {
    "action": mt5.TRADE_ACTION_DEAL,
    "symbol": symbol,
    "volume": 0.01,          # 0.01 迷你手
    "type": mt5.ORDER_TYPE_BUY,
    "price": tick.ask,
    "sl": tick.ask - 500 * point,  # 止损 500 点
    "tp": tick.ask + 1000 * point, # 止盈 1000 点
    "deviation": 20,         # 滑点容忍 20 点
    "magic": 123456,         # 策略编号，便于区分
    "comment": "swing_signal_001",
    "type_time": mt5.ORDER_TIME_GTC,
    "type_filling": mt5.ORDER_FILLING_IOC,
}
result = mt5.order_send(request)
if result.retcode == mt5.TRADE_RETCODE_DONE:
    print(f"✅ 订单成交: {result.order}")
else:
    print(f"❌ 订单失败: {result.comment} (代码: {result.retcode})")

# 6. 获取持仓
positions = mt5.positions_get()
for pos in positions:
    print(f"{pos.symbol} | {pos.volume}手 | 浮动盈亏: ${pos.profit}")

# 7. 用完断开
mt5.shutdown()
```

**自动化部署要点：**

- MT5 需要在你本地或云服务器上保持运行（最小化到系统托盘即可），不能退出
- 建议在 $5-15/月的云服务器（Vultr / DigitalOcean / 阿里云海外节点）上运行
- 你的策略脚本通过 `mt5` 库连接运行中的 MT5 发送指令
- MT5 会自动定期刷新市场数据并保持连接

**安全建议：**
- MT4/5 的交易密码设置**只读密码**（投资密码）做数据查看，用**交易密码**给自动化脚本
- 不要将你的主账户密码放在版本控制仓库中（用 `.env` 配置文件）

### 4.4 安全机制——Kill Switch

这是最关键的保障。无论你用哪层自动化，必须有：

```python
# kill_switch.py
# 在 main.py 最前面调用

import os

KILL_SWITCH_FILE = "STOP_TRADING.flg"

def check_kill_switch():
    """如果存在 STOP_TRADING.flg 文件，立即停止所有交易操作"""
    if os.path.exists(KILL_SWITCH_FILE):
        print("🛑 Kill Switch 触发！停止所有交易操作")
        # 清空所有挂单
        # 不执行任何新交易
        # 发通知给你
        return True
    return False

def enable_kill_switch():
    """创建一个文件即可，可以手动创建"""
    open(KILL_SWITCH_FILE, 'w').close()
    
def disable_kill_switch():
    """删除文件即恢复"""
    if os.path.exists(KILL_SWITCH_FILE):
        os.remove(KILL_SWITCH_FILE)
```

使用方式：
- **手动**：在量化目录下新建一个空文件 `STOP_TRADING.flg`，系统下次运行就会全部停止
- **远程**：通过 Telegram Bot、微信、或任何消息工具远程触发文件创建
- **自动**：当日亏损超过阈值时，系统自己创建

### 6.4 你应该避免的陷阱

1. **过拟合是 #1 杀手。** 你的信号在回测上看起来越好，越可疑。
2. **不要把完全自动下单和真实市场连接。** 在未经数月的 paper trading 验证前，保持人工审核环节。
3. **不要每天看账户。** Swing 策略看周线就够了——每天看会增加焦虑，导致你过早撤单或频繁调整。
4. **不要追逐热门策略。** "用 GPT 选股"、"深度学习高频" 在 YC 新闻里很多，但多数带不来超额收益。
5. **关于黄金（GLD）的特殊性：** 黄金与股票相关性低，适合作为对冲，但 GLD 的走势主要受实际利率和美元影响，股票上常用的动量因子效果不如预期。建议单独为 GLD 设计简单的技术面规则（200 周均线以上持仓，以下观望）。
